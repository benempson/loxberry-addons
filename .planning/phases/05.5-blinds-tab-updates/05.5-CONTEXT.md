# Phase 05.5: Blinds tab updates - Context

**Gathered:** 2026-03-17
**Status:** Ready for planning
**Source:** User requirements (conversation)

<domain>
## Phase Boundary

This phase delivers three UI improvements to the Blinds tab:
1. Add a Description column after Device Name, before Position
2. Make all columns sortable (click header to sort, click again to toggle asc/desc)
3. Add 20px gap between tabs navbar and Blinds table

</domain>

<decisions>
## Implementation Decisions

### Description Column
- Add a Description column after Device Name and before Position
- Uses the same device description data already available in $row['description']

### Sortable Columns
- All Blinds tab columns must be sortable by clicking the header
- Click toggles between ascending and descending sort order
- Use the same sortTable() function already used on the Device Status tab

### Tab-Table Gap
- Add 20px gap/margin between the tabs navbar and the Blinds table

### Claude's Discretion
- Whether to add sort type hints (str vs num) per column
- CSS approach for the 20px gap (margin-top on table vs padding on container)

</decisions>

<specifics>
## Specific Ideas

- The Device Status tab already has sortable columns using sortTable() with onclick handlers
- The Blinds tab was added in Phase 5.4 but without sorting support
- Device description is already available in $table_rows as $row['description']
- The same sortTable() JS function can be reused for the Blinds tab

</specifics>

## Deferred Ideas

None — all items are in scope for this phase.
