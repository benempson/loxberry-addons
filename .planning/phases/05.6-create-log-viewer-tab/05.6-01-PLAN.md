Create `bin/lib/logger.js` with these exact exports and behavior:

```javascript
module.exports = { init, log, SEVERITIES };
```

**SEVERITIES constant:** `['Debug', 'Info', 'Warning', 'Error']`

**`init(filePath, opts = {})`:**
- Sets module-level `logPath = filePath`
- If `opts.maxSize` provided (number, bytes), sets `maxSize` (default `1048576` = 1MB)
- If `opts.maxFiles` provided (number), sets `maxFiles` (default `5`)

**`rotate()` (internal, not exported):**
- If `logPath` is null, return
- Try `fs.statSync(logPath)` -- if file doesn't exist (ENOENT), return
- If `stat.size < maxSize`, return (no rotation needed)
- Loop from `maxFiles - 1` down to `1`:
  - `from` = `i === 1 ? logPath : logPath + '.' + (i - 1)`
  - `to` = `logPath + '.' + i`
  - Try `fs.renameSync(from, to)`, catch and ignore (file may not exist)
- After loop, `fs.writeFileSync(logPath, '')` to create fresh empty log file

**`log(severity, source, message)`:**
- If `logPath` is null, return (logger not initialized)
- If `severity` not in SEVERITIES, default to `'Info'`
- Call `rotate()` before writing
- Build JSON entry: `{ ts: new Date().toISOString(), sev: severity, src: source, msg: message }`
- Append `JSON.stringify(entry) + '\n'` to logPath using `fs.appendFileSync(logPath, entry, 'utf8')`

This ensures `readConfig()` automatically picks up LOGGING values from the INI file and coerces them to numbers.
