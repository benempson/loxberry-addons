# Phase 05.6: Create Log Viewer Tab - Context

**Gathered:** 2026-03-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a "Logs" tab to the plugin web UI showing watchdog activity logs. This requires both a log-writing mechanism (watchdog.js currently sends all output to /dev/null) and a UI to display, filter, and auto-refresh log entries. Does NOT add new monitoring capabilities or change watchdog behavior beyond adding structured logging output.

</domain>

<decisions>
## Implementation Decisions

### Log Content & Scope
- Log everything: watchdog run results, cron registration events, config saves, notification delivery results, errors
- Each log entry tagged with severity: Debug, Info, Warning, Error
- Structured log format with timestamp, severity, source (e.g., watchdog, cron, config, notify), and message

### Log Retention & Rotation
- 1MB max per log file, rotate when exceeded (configurable in Settings tab)
- Keep max 5 rotated log files, size-based rotation (configurable in Settings tab)
- Log viewer reads across all rotated files when applying filters
- Settings tab gets a new "Logging" section with max file size and max rotated files fields

### Log Display Format
- Structured table with columns: Timestamp | Severity | Source | Message
- Severity shown as colored badge (grey Debug, green Info, yellow Warning, red Error) AND subtle row background tint
- Long messages truncated to one line with click-to-expand
- Newest entries first (top of table)

### Filter Controls
- Dropdown filter for log volume (capitalize correctly):
  - 100 Lines, 200 Lines, 300 Lines
  - Last 1 Hour, Last 2 Hours, Last 4 Hours, Last 8 Hours, Last 24 Hours, Last 48 Hours, Last 96 Hours
  - All Logs
- Dropdown filter for severity: All, Debug, Info, Warning, Error
- Text filter: free-text search box that applies a substring match (like SQL `%text%`) across the message column
- Default view: Last 100 Lines, All severities, no text filter

### Log Freshness
- Auto-poll for new entries (reuse Phase 5.4 AJAX polling pattern)
- New log entries that arrive via polling get a brief highlight animation (yellow/green fade-out after 2-3 seconds)
- No manual refresh button needed (auto-poll handles it)
- No "Clear logs" button (retention handles cleanup)
- No "Download logs" button (view-only in UI)

### Claude's Discretion
- Poll interval for log updates
- Exact highlight animation implementation
- Log file path and naming convention within Loxberry data directory
- How to redirect watchdog.js console output to structured log file
- PHP endpoint for serving filtered log data via AJAX
- Table pagination or virtual scrolling approach for large log sets

</decisions>
