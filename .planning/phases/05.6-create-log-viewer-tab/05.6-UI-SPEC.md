# Phase 05.6 — UI Design Contract

> Visual and interaction contract for the Logs tab. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | jQuery Mobile 1.4.x (bundled by Loxberry SDK) |
| Icon library | none (text labels only, matching existing tabs) |
| Font | System default (inherited from Loxberry SDK / jQuery Mobile theme) |

**Note:** This is a Loxberry PHP plugin. All styling uses inline CSS to match the established pattern across Settings, Device Status, and Blinds tabs. No external CSS frameworks, no build step.

**Primary focal point:** The log table, which occupies the majority of the tab content area and is the central element users interact with.

## Component Inventory

### 1. Logs Tab (navbar entry)

```
$navbar[4]['Name'] = $L['NAV.LOGS'];
$navbar[4]['URL']  = 'index.php?tab=logs';
```

Tab index: 3 (zero-based: settings=0, status=1, blinds=2, logs=3).

### 2. Filter Controls Row

Layout: `display:flex; align-items:center; gap:16px; margin:16px 0; flex-wrap:wrap;`

Contains three controls in a single row:

| Control | Type | HTML Element | Default Value |
|---------|------|-------------|---------------|
| Volume filter | Dropdown | `<select>` with jQuery Mobile enhancement | "100" (100 Lines) |
| Severity filter | Dropdown | `<select>` with jQuery Mobile enhancement | "all" (All) |
| Text search | Text input | `<input type="search">` | Empty |

#### Volume Dropdown Options

| Label | Value |
|-------|-------|
| 100 Lines | `100` |
| 200 Lines | `200` |
| 300 Lines | `300` |
| Last 1 Hour | `1h` |
| Last 2 Hours | `2h` |
| Last 4 Hours | `4h` |
| Last 8 Hours | `8h` |
| Last 24 Hours | `24h` |
| Last 48 Hours | `48h` |
| Last 96 Hours | `96h` |
| All Logs | `all` |

#### Severity Dropdown Options

| Label | Value |
|-------|-------|
| All | `all` |
| Debug | `debug` |
| Info | `info` |
| Warning | `warning` |
| Error | `error` |

#### Text Search Input

- Placeholder: "Search logs..."
- Minimum characters: 2 (debounce 350ms, matching Device Status search)
- Triggers AJAX reload on input (after debounce)

### 3. Log Table

| Column | Width | Sortable | Content |
|--------|-------|----------|---------|
| Timestamp | auto | No | ISO timestamp formatted as `YYYY-MM-DD HH:MM:SS` (local time) |
| Severity | 80px max | No | Colored badge pill |
| Source | 80px max | No | Plain text (e.g., watchdog, cron, config, notify) |
| Message | remaining | No | Truncated to one line; click to expand |

Table styles (matching existing tables):
- `width:100%; border-collapse:collapse;`
- Header row: `background:#f5f5f5;`
- Header cells: `padding:8px; text-align:left;`
- Body rows: `border-bottom:1px solid #ddd;`
- Body cells: `padding:8px;`
- Severity badge: `padding:4px 8px; border-radius:4px; font-size:0.85em; color:#fff;`
- Row background tint applied via inline style on `<tr>` element

### 4. Message Expand/Collapse

- Default: `white-space:nowrap; overflow:hidden; text-overflow:ellipsis; max-width:400px;`
- Expanded: `white-space:pre-wrap; word-wrap:break-word; max-width:none;`
- Toggle: Click on the message cell toggles between states
- Cursor: `cursor:pointer;` on message cells
- Visual cue: No icon needed; ellipsis indicates truncation

### 5. Entry Count Notice

- Position: Below filter controls, above log table
- Style: `font-size:0.85em; color:#666; margin:4px 0;`
- Copy pattern: "Showing 100 of 342 entries" or "Showing 42 entries"
- Truncation notice: "Showing most recent 5,000 entries (log limit reached)" when server caps at 5000

### 6. Data Refreshed Timestamp

- Position: Below filter controls, same line as entry count (right-aligned via flex spacer)
- Style: `font-size:0.85em; color:#666;` (matches Device Status and Blinds "Data refreshed" pattern)
- Format: "Data refreshed: YYYY-MM-DD HH:MM:SS"
- Updated on each successful poll

### 7. New Entry Highlight Animation

CSS keyframe animation applied to `<tr>` elements for entries newer than the previous poll's newest timestamp.

```css
@keyframes log-highlight {
  from { background-color: rgba(255, 235, 59, 0.4); }
  to { background-color: transparent; }
}
.log-new {
  animation: log-highlight 2.5s ease-out forwards;
}
```

After animation completes (2.5s), the `.log-new` class is removed (or left to settle at `transparent`). The row background then shows only the severity tint.

### 8. Settings Tab -- Logging Section

New section added below existing settings sections.

Section heading: `<h3>` with text from `$L['SETTINGS.SECTION_LOGGING']`

| Field | Type | Options | Default |
|-------|------|---------|---------|
| Max Log File Size | `<select>` dropdown | 512 KB, 1 MB, 2 MB | 1 MB (value: 1024) |
| Max Rotated Files | `<select>` dropdown | 3 files, 5 files, 10 files | 5 files (value: 5) |

Uses `data-role="fieldcontain"` wrapper matching existing settings fields.

## Interaction Contracts

### Polling Behavior

| Property | Value |
|----------|-------|
| Poll interval | 10,000ms (10 seconds) |
| Active condition | Logs tab is active (tab index 3) AND document is not hidden |
| Inactive behavior | No log polling when Settings, Status, or Blinds tabs are active |
| Device polling | Must skip when Logs tab is active (update `activeTab === 0` check to also skip index 3) |
| Failure handling | Silent fail, retry on next interval (matches existing pattern) |
| AJAX method | `jQuery.post('index.php', { action: 'get_log_data', volume: ..., severity: ..., search: ... })` |

### Filter Interaction

| Action | Behavior |
|--------|----------|
| Change volume dropdown | Immediate AJAX reload (no debounce) |
| Change severity dropdown | Immediate AJAX reload (no debounce) |
| Type in search box | AJAX reload after 350ms debounce, minimum 2 characters |
| Clear search box | Immediate AJAX reload (all entries matching other filters) |
| Filters persist across polls | Yes -- poll reuses current filter values |

### Tab Activation

| Trigger | Action |
|---------|--------|
| Click Logs tab | Load log data via AJAX with default filters; start log polling |
| Leave Logs tab | Stop log polling; start device polling (if going to Status/Blinds) |
| URL parameter `?tab=logs` | Activate Logs tab on page load via `jQuery('[data-role="tabs"]').tabs("option", "active", 3)` |

## Empty State

When no log entries exist (fresh install, watchdog never run):

- Display centered within the tab content area
- Text: "No log entries" (bold, normal size) followed by "Logs will appear here after the watchdog runs." (0.85em, color #666)
- No icon, no illustration (matches the minimal existing UI style)

When log entries exist but filters match nothing:

- Display in place of the table body
- Single row spanning all columns: "No entries match the current filters."
- Style: `text-align:center; color:#666; padding:24px 8px;`

---

## Error State

When AJAX log fetch fails:

- Do not clear the existing table (keep stale data visible)
- Show error notice below filter controls: "Failed to load logs. Will retry automatically."
- Style: `font-size:0.85em; color:#f44336; margin:4px 0;`
- Notice auto-clears on next successful poll
