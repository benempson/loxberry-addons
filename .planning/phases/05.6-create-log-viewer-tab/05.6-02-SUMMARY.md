## Accomplishments
- Logs tab with filter controls (volume dropdown, severity dropdown, text search) and log table with Timestamp/Severity/Source/Message columns
- AJAX `get_log_data` endpoint reads JSONL from watchdog.log and rotated files, applies server-side filtering, returns sorted JSON
- Auto-polling every 10s when Logs tab is active; new entries highlighted with yellow fade animation (2.5s)
- Settings tab Logging section with max file size (512KB/1MB/2MB) and max rotated files (3/5/10) dropdowns
- Config saves produce JSONL log entries in watchdog.log
- Device polling suppressed on Logs tab; log polling suppressed on other tabs

## Decisions Made
- Logs tab is navbar index 4 (zero-based tab index 3) -- placed after Blinds as the 4th tab
- Log polling runs at 10s (faster than device polling at 30s) since log data is lightweight
- Server-side filtering keeps client simple -- AJAX returns pre-filtered, pre-sorted entries
- 5000-entry hard cap prevents memory issues on large log files
- Config save log entry uses PHP file_put_contents with LOCK_EX for safe concurrent append
