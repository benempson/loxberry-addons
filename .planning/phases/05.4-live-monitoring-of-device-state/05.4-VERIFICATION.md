### Human Verification Required

The following items cannot be confirmed programmatically and require manual testing on a Loxberry host with live Zigbee2MQTT data:

#### 1. LQI values display correctly from live state.json

**Test:** Open the plugin UI on a Loxberry host with Z2M running. Navigate to Device Status.
**Expected:** LQI column shows numeric values (e.g., 142, 255) for devices that have recently communicated; shows "N/A" for devices with no Z2M state entry.
**Why human:** Requires live Z2M state.json with `linkquality` fields populated.

#### 2. Blinds tab filter works with real device names

**Test:** If MS-108ZR devices exist in the registry, navigate to the Blinds tab.
**Expected:** Only devices whose friendly_name starts with "MS-108ZR" appear. Position shows numeric value with "%" suffix. State and Motor Reversal show current Z2M state values.
**Why human:** Requires live devices with that naming convention; behavior with zero matching devices (empty table) should also be confirmed acceptable.

#### 3. AJAX polling updates are visible to user

**Test:** Leave the Device Status tab open for 30+ seconds. Trigger a Z2M state change (e.g., a device goes offline or battery changes).
**Expected:** The table row updates within ~30 seconds without any page action.
**Why human:** Requires live Z2M state changes to observe the poll cycle in action.

#### 4. Polling correctly skips the Settings tab

**Test:** Navigate to the Settings tab and observe network requests for 60+ seconds.
**Expected:** No `index.php` POST requests with `action=get_status_data` are made while Settings tab is active.
**Why human:** Requires browser devtools network tab observation.

#### 5. Exclude checkbox survives live update

**Test:** Check/uncheck an exclude checkbox. Wait for next 30s poll cycle to complete.
**Expected:** The checkbox state (checked or unchecked) reflects the persisted exclusion state from the server-returned JSON, not the pre-update DOM state.
**Why human:** Requires timing the interaction with the poll cycle.
