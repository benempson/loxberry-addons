Three changes to index.php to add device state tooltips:

**1. PHP: Add data-z2m-state attribute to alert badge span (around line 746)**

In the table row loop (foreach $table_rows), look up each device's Z2M state from the already-loaded `$z2m_state_data` array using `$row['name']` (friendly_name key). Encode it as an HTML-safe data attribute on the alert badge `<span>`:

```php
$device_z2m_state = isset($z2m_state_data) && is_array($z2m_state_data) && isset($z2m_state_data[$row['name']]) ? $z2m_state_data[$row['name']] : null;
$state_json_attr = $device_z2m_state ? htmlspecialchars(json_encode($device_z2m_state), ENT_QUOTES) : '';
```

Add `data-z2m-state="<?php echo $state_json_attr; ?>"` and `style="cursor:pointer;"` to the existing alert badge `<span>` element. Only add cursor:pointer if $state_json_attr is non-empty.

**2. HTML: Add a single shared jQuery Mobile popup container**

Place this BEFORE the closing `</div>` of `#tab-status` (before line 758), OUTSIDE the `<?php endif; ?>` block -- put it just before the `</div>` for tab-status:

```html
<div data-role="popup" id="state-popup" class="ui-content" data-theme="a"
     style="max-width:420px;max-height:350px;overflow:auto;">
    <pre id="state-popup-content" style="margin:0;font-size:0.8em;white-space:pre-wrap;word-wrap:break-word;"></pre>
</div>
```

**3. JS: Add click handler to open popup with device state**

Add this in the `<script>` section, inside the DOMContentLoaded handler or as a standalone wired function:

```javascript
// Z2M state tooltip popup
if (typeof jQuery !== 'undefined') {
    jQuery(document).on('click', '[data-z2m-state]', function(e) {
        e.preventDefault();
        e.stopPropagation();
        var raw = this.getAttribute('data-z2m-state');
        if (!raw) return;
        try {
            var state = JSON.parse(raw);
            jQuery('#state-popup-content').text(JSON.stringify(state, null, 2));
            jQuery('#state-popup').popup('open', { positionTo: this });
        } catch(ex) {}
    });
}
```

IMPORTANT: The popup div MUST be in the static HTML (PHP output), not dynamically created, so jQuery Mobile enhances it on pagecreate. Use a single shared popup element and swap content via JS -- do NOT create one popup per device row.

NOTE: `$z2m_state_data` is already loaded around lines 367-370 when Z2M path is configured. If Z2M path is not configured or state.json is missing, it won't be set -- the isset() check handles this gracefully.

**2. JS: Fix clear search button not triggering filter**

In the `wireSearch()` function (around line 809), add a `search` event listener alongside the existing `input` listener. The `search` event fires when the user clicks the browser's native clear button (the X icon) on `input[type=search]`:

```javascript
el.addEventListener('search', function() {
    if (searchTimer) clearTimeout(searchTimer);
    applyFilters();
});
```

Add this right after the existing `el.addEventListener('input', ...)` call (after line 815). This ensures that when the clear button is clicked, the filter is immediately re-applied (no debounce delay needed for clear).
