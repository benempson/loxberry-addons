# Phase 05.3: Display device state in tooltip on device status page - Context

**Gathered:** 2026-03-17
**Status:** Ready for planning
**Source:** User requirements (conversation)

<domain>
## Phase Boundary

This phase delivers three UI fixes/features on the device status page:
1. Device state tooltip showing Z2M raw JSON state
2. Search placeholder text alignment fix
3. Clear search button functionality fix

</domain>

<decisions>
## Implementation Decisions

### Device State Tooltip
- Show the current Z2M state (equivalent to what's visible in Z2M's State tab) as raw JSON
- Display on hover for desktop, click/tap for mobile
- Triggered from the Alert Status entry/cell in the device table
- Plain JSON data is sufficient (no formatting beyond readability needed)

### Search Placeholder Fix
- "Filter devices" placeholder text is left-aligned and overlaps the magnifying glass icon
- Text needs padding/offset so it doesn't sit behind the icon

### Clear Search Bug Fix
- The clear search icon (×) currently clears the textbox value but doesn't trigger the search/filter
- User must manually delete text for the filter to actually reset
- Clicking the clear icon should both clear the text AND trigger the filter to show all devices

### Claude's Discretion
- Tooltip implementation approach (CSS tooltip, popup, jQuery Mobile popup widget)
- JSON formatting in tooltip (pretty-print vs compact)
- How Z2M state data is passed to the frontend (inline data attribute, AJAX call, etc.)
- CSS specifics for search placeholder padding

</decisions>

<specifics>
## Specific Ideas

- Z2M state data is already available server-side (z2m-reader.js / PHP reads state.json)
- The device status page already has device data rendered in a table
- jQuery Mobile is the UI framework in use
- Search functionality exists but the clear button event handler is incomplete

</specifics>

<deferred>
## Deferred Ideas

None — all items are in scope for this phase.

</deferred>
