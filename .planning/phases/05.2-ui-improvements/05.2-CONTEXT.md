# Phase 5.2: UI Improvements - Context

**Gathered:** 2026-03-17
**Status:** Ready for planning

<domain>
## Phase Boundary

Consolidate exclusion management into the Device Status tab and improve its usability. Remove the standalone Exclusions tab. Add search, description column, and inline exclude toggles to the Device Status tab.

</domain>

<decisions>
## Implementation Decisions

### Remove Exclusions Tab
- Remove the standalone Exclusions tab entirely
- The tab bar becomes: Settings | Device Status (two tabs instead of three)

### Inline Exclusion on Device Status Tab
- Add an "Exclude" column with a checkbox on each device row
- Checking/unchecking saves the exclusion state (write IEEE to EXCLUSIONS.devices in INI)
- Add a "Show Excluded" toggle at the top of the Device Status tab, default unchecked
- When unchecked, excluded devices are hidden from the table
- When checked, excluded devices appear in the table (with their Excluded badge)

### Search on Device Status Tab
- Move the search/filter box to the Device Status tab (was on the Exclusions tab)
- Filters the device table by device name as you type

### Description Column
- Add a Description column to the Device Status table
- Source: devices.yaml `description` field (already available via readZ2mDevices)
- Shows the user-assigned description from zigbee2mqtt (e.g., "Habitació 1 P2")

### Claude's Discretion
- Exact placement of Show Excluded toggle and search box
- How exclude checkbox save works (AJAX vs form submit vs auto-save)
- Column ordering in the table
- Whether description column is sortable

</decisions>
