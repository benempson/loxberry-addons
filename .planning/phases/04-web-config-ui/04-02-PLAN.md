- Key: EXCLUSIONS.devices
- Value: comma-separated IEEE addresses, NO inline # comments
- Example: `devices=0x00158d0001a2b3c4,0x00158d0002b3c4d5`
- Friendly names displayed from state.json, NOT stored in INI

  <name>Task 2: Device Status tab with sortable table</name>
  <files>webfrontend/htmlauth/index.php</files>
  <action>
    Replace the #tab-status placeholder in index.php with the Device Status tab:

    1. **POST handler for refresh** (add to POST handler block):
       - If `$_POST['action'] === 'refresh'`:
         - `exec('node ' . LBPBINDIR . '/watchdog.js 2>&1', $output, $retval)`
         - Redirect to `index.php?tab=status&msg=` . ($retval === 0 ? 'refresh_ok' : 'refresh_fail')

    2. **PHP data preparation** (reuses $devices and $state loaded for Exclusions tab):
       - Create `$table_rows` array, for each device compute:
         - `name`: friendly_name (htmlspecialchars)
         - `last_seen_age`: human-readable age string using a `formatAge($seconds)` helper function. If last_seen is null/empty, show "Never". Use `strtotime()` to parse ISO date, compute diff from `time()`.
         - `last_seen_ts`: raw Unix timestamp for sort data attribute
         - `battery`: battery value or 'N/A' for mains-powered (check power_source)
         - `alert_status`: derive from alerts object. If offline.active: "Offline". If battery.active: "Low Battery". If device IEEE in excluded list: "Excluded". Otherwise: "OK".
         - `sort_priority`: 0 for offline alerts, 1 for battery alerts, 2 for excluded, 3 for OK (for default sort)
       - Sort $table_rows by sort_priority ASC, then name ASC

    3. **Tab content HTML**:
       - Flash messages for refresh_ok / refresh_fail
       - "Last updated" timestamp from `$state['last_run']` (formatted with `date()`, or "Never" if null)
       - Refresh Data form: `<form method="post"><input type="hidden" name="action" value="refresh"><button type="submit">Refresh Data</button></form>`
       - If `$state_missing`: show "No data yet" message with suggestion to click Refresh Data
       - Otherwise: `<table id="device-table" data-role="table" class="ui-responsive">` with:
         - `<thead>`: clickable column headers (Name, Last Seen, Battery, Status) -- each `<th>` has `onclick="sortTable(this.closest('table'), N, 'type')"` where N is column index and type is 'str' or 'num'
         - `<tbody>`: one `<tr>` per device with:
           - Name cell
           - Last Seen cell with `data-sort-value="{unix_ts}"` for numeric sort
           - Battery cell with `data-sort-value="{battery_int}"` (use -1 for N/A so mains devices sort to bottom)
           - Status cell with appropriate badge styling: red background for "Offline", orange for "Low Battery", grey for "Excluded", green for "OK". Use inline style or jQuery Mobile classes. `data-sort-value="{sort_priority}"`
       - Tab activation: `?tab=status` sets active tab index to 2

    4. **Inline JS** (add to script block):
       - `sortTable(table, col, type)` function: vanilla JS table sort. Reads `data-sort-value` from cells (falls back to textContent). Toggles ascending/descending on repeated clicks. Uses `table.dataset.sortCol` and `table.dataset.sortAsc` to track state.

    5. **PHP helper function** `formatAge($seconds)`: Returns "just now" (<60s), "Xm ago" (<1h), "Xh ago" (<24h), "Xd ago" (>=24h). Define near the top of the PHP file.
