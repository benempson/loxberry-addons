<feature>
  <name>evaluateDevices -- threshold evaluation with state machine transitions</name>
  <files>bin/lib/evaluator.js, tests/evaluator.test.js</files>
  <behavior>
    ## Function Signature
    `evaluateDevices(state, config, now)` -- mutates state.devices[ieee].alerts in place, returns { transitions, summary, excludedCount }

    ## Offline Evaluation (ALRT-01, DEVT-02)
    - Device with last_seen older than offline_hours threshold: ok-to-alert transition, set alerts.offline=true, alerts.offline_sent_at=now
    - Device with null last_seen: immediate offline flag with detail "never seen" (not "last seen X hours ago")
    - Device seen within threshold: no change if ok; if previously alert, recovery transition (alerts.offline=false, alerts.offline_recovered_at=now)
    - Already-alert device still beyond threshold: no transition emitted (ALRT-03 duplicate suppression)

    ## Battery Evaluation (ALRT-02, DEVT-03)
    - Battery-powered device (power_source === 'Battery') with battery <= battery_pct: ok-to-alert, set alerts.battery=true, alerts.battery_sent_at=now
    - Battery-powered device with null battery: NO alert triggered
    - Non-battery device: skip battery evaluation entirely (even if battery field exists)
    - Recovery: battery must exceed battery_pct + 5 (hysteresis) to clear, set alerts.battery=false, alerts.battery_recovered_at=now
    - Battery at exactly threshold + hysteresis (e.g., 30 when threshold=25): stays in alert (strict greater-than)
    - Already-alert device still low: no transition emitted (ALRT-03)

    ## Exclusions (ALRT-05)
    - Devices matching exclusion list (by IEEE or friendly_name, case-insensitive) are skipped entirely
    - No state mutations for excluded devices
    - Excluded count tracked in result

    ## Recovery (ALRT-04)
    - Offline recovery: device seen again within threshold
    - Battery recovery: battery > threshold + hysteresis
    - Recovery transitions emitted alongside alert transitions

    ## Transition Event Shape
    - { type: 'offline'|'battery', transition: 'alert'|'recovery', ieee, friendly_name, detail, timestamp }
    - detail for offline alert: "not seen for X.X hours" or "never seen"
    - detail for battery alert: "battery at X% (threshold: Y%)"
    - detail for offline recovery: "seen again after X.X hours offline"
    - detail for battery recovery: "battery recovered to X% (was below Y%)"

    ## Summary Shape
    - { total_devices, excluded, evaluated, alerts: { offline, battery, total }, transitions: { new_alerts, recoveries } }

    ## State Mutations
    - state.last_evaluation = now.toISOString()
    - state.evaluation_summary = summary object
    - state.pending_notifications = [...existing, ...new transitions]
    - Per-device alerts object extended with offline_recovered_at, battery_recovered_at (handle missing fields from Phase 1 state gracefully)

    ## Edge Cases
    - Empty state.devices: returns zero counts, no transitions
    - Legacy state without recovered_at fields: treat as null
    - Corrupt/missing config.THRESHOLDS: use defaults (offline_hours=24, battery_pct=25)
  </behavior>
